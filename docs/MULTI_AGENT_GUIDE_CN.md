# 多 Agent 操作手册

本文档聚焦 Web UI 中“团队管理 / 多 Agent 管理”的日常操作，适合需要长期维护多个 Agent 的用户。

## 1. 适用范围

在当前项目里：

- 每个 Agent 都是独立实例
- 每个 Agent 都有自己的独立工作区
- 每个 Agent 都可以使用不同的 provider、model、权限档位与能力标签
- 每个 Agent 都有独立的 `SOUL.md` 与 `USER.md`

团队管理负责：

- 创建和编辑 Agent
- 将 Agent 编入团队
- 定义团队内主次、职责和优先级

Configuration 页面只保留全局默认值，不再承担 Agent 实例级模型配置。

## 2. 每个 Agent 的核心档案

Agent 工作区内最重要的两个文件是：

- `SOUL.md`
  作用：定义该 Agent 的身份定位、职责重点、沟通风格、边界约束
- `USER.md`
  作用：记录用户偏好、协作约定、特殊说明

它们共同决定：

- Agent 如何理解自己的角色
- Agent 如何和你说话
- Agent 在什么情况下可以主动执行、接力或停下来确认

## 3. 创建 Agent 的推荐流程

推荐做法是先最小化创建，再在第一次私聊中完成具体定制。

### 第一步：创建最小实例

创建时只需要优先确认：

- Agent 名称
- provider
- model
- 工作区路径
- 权限档位
- 能力标签

### 第二步：进入私聊完成首次引导

首次私聊时，Agent 会根据当前状态继续补全：

- 身份定位
- 职责重点
- 沟通风格
- 边界约束
- 用户偏好

如果引导完成并落盘成功，待配置状态会自动消失。

## 4. 两种配置方式

在多 Agent 详情页里，建议把调整方式理解为两层：

### 方式 A：编辑“配置摘要”

适合：

- 快速补充或修正关键信息
- 日常维护
- 不想直接改 Markdown 结构

“配置摘要”当前分为五类：

- 身份定位
- 职责重点
- 沟通风格
- 边界约束
- 用户偏好

使用方式：

1. 每行填写一条
2. 点击“保存摘要”
3. 系统自动把内容写回 `SOUL.md` / `USER.md` 对应章节

### 方式 B：直接编辑 `SOUL.md` / `USER.md`

适合：

- 你希望完全控制 Markdown 内容
- 你需要补充更长的上下文
- 你想保留自己习惯的文档结构

这两种方式可以混合使用，但建议先用“配置摘要”维护结构化关键信息，再在文件里补充更长说明。

## 5. 配置摘要如何回写

当前回写规则如下：

- `身份定位` -> `SOUL.md / ## 身份定位`
- `职责重点` -> `SOUL.md / ## 职责重点`
- `沟通风格` -> `SOUL.md / ## 沟通风格`
- `边界约束` -> `SOUL.md / ## 边界约束`
- `用户偏好` -> `USER.md / ## 用户偏好`

保存摘要时还会做两件事：

- 清理首次引导的待配置标记
- 重新生成页面中的结构化摘要显示

## 6. 团队中的主次关系

当前项目里不再强调“全局主 Agent”。

更合理的理解是：

- Agent 是平级实例
- 团队里才定义主次关系和职责分工

因此建议：

- 在 Agent 维度配置实例能力、模型、权限和档案
- 在 Team 维度配置 lead、职责、优先级、协作顺序

## 7. 建议的档位策略

如果你维护多个 Agent，建议按职责分层：

- 协调型 Agent
  推荐更强的接力与任务能力，强调拆解、同步和终止条件
- 工程型 Agent
  推荐更强的文件、终端、浏览器能力，强调验证和风险边界
- 研究型 Agent
  推荐保留网页检索与总结能力，强调证据、结论和不确定性区分
- 陪伴型 Agent
  推荐较保守权限，强调语气、节奏和解释方式

## 8. 常见问题

### 8.1 为什么我保存了配置摘要，但 `SOUL.md` / `USER.md` 也变了

这是预期行为。

“配置摘要”不是独立存储，而是结构化编辑层。保存后会直接回写到对应 Markdown 文件。

### 8.2 为什么首次私聊后待配置状态会消失

因为 Agent 已经把正式内容写回档案，并移除了待配置标记，说明初始化完成。

### 8.3 如果我只想改一句话，应该改哪里

优先改“配置摘要”。只有当你需要更复杂的上下文时，再直接编辑 `SOUL.md` 或 `USER.md`。

### 8.4 如何验证这条链路没坏

可以直接运行：

```bash
./horbot.sh smoke agent-assets
```

它会验证：

- Agent 详情页可打开
- `SOUL.md` / `USER.md` 可保存
- “配置摘要”可保存
- 刷新后内容仍然能回显
