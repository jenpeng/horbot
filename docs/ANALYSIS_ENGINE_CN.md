# 内容分析引擎规范文档

## 1. 概述

本文档定义了内容分析引擎的规范规则，包括分析类型、处理流程、算法标准和输出格式。本规范适用于从各类文件中提取的内容进行深度分析，确保分析结果的准确性、一致性和可复用性。

### 1.1 设计目标

- **准确性**：提供精确的内容分析结果，支持多种分析维度
- **可扩展性**：支持自定义分析规则和算法，便于功能扩展
- **性能高效**：优化分析算法，降低计算资源消耗
- **结果一致**：确保相同内容在不同环境下产生一致的分析结果

### 1.2 适用范围

本规范适用于以下场景：
- 文本内容提取与分析
- 文档结构解析
- 关键词提取与主题识别
- 情感分析
- 实体识别
- 代码分析

## 4. 算法定义

### 4.1 关键词提取算法

#### 4.1.1 TF-IDF算法

**算法原理**：
- TF（Term Frequency）：词频，计算词语在文档中出现的频率
- IDF（Inverse Document Frequency）：逆文档频率，降低常见词权重

**计算公式**：

```
TF(t) = 词语t在文档中出现的次数 / 文档总词数
IDF(t) = log(语料库文档总数 / 包含词语t的文档数)
TF-IDF(t) = TF(t) × IDF(t)
```

**算法参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| top_k | 10 | 返回前k个关键词 |
| min_word_length | 2 | 最小词语长度 |
| use_idf | true | 是否使用IDF权重 |
| smooth_idf | true | 是否平滑IDF值 |

#### 4.1.2 TextRank算法

**算法原理**：
- 基于图排序的关键词提取算法
- 将文本分为候选词窗口，构建词图
- 通过迭代计算词的重要性

**算法参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| window_size | 5 | 共现窗口大小 |
| top_k | 10 | 返回前k个关键词 |
| damping_factor | 0.85 | 阻尼系数 |
| max_iterations | 100 | 最大迭代次数 |

### 4.2 文本摘要算法

#### 4.2.1 抽取式摘要

**算法原理**：
- 识别文档中的关键句子
- 基于句子权重排序选择重要句子
- 组合形成摘要

**关键句子评分因素**：

| 因素 | 权重 | 说明 |
|------|------|------|
| 位置权重 | 0.2 | 句子在文档中的位置（首尾更重要） |
| 长度权重 | 0.1 | 句子长度适中为佳 |
| 关键词匹配 | 0.3 | 包含关键词的数量 |
| 相似度惩罚 | 0.2 | 与已选句子的重复度 |
| 标题相似度 | 0.2 | 与文档标题的相似度 |

**算法参数**：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| summary_ratio | 0.2 | 摘要长度与原文比例 |
| min_sentence_length | 20 | 最小句子长度 |
| max_sentence_length | 200 | 最大句子长度 |

#### 4.2.2 生成式摘要（可选）

使用预训练语言模型生成摘要，需要额外配置：

| 参数 | 说明 |
|------|------|
| model_name | 摘要模型名称 |
| max_length | 生成摘要最大长度 |
| temperature | 生成随机性控制 |

### 4.3 情感分析算法

#### 4.3.1 基于词典的情感分析

**词典组成**：

| 词典类型 | 说明 |
|----------|------|
| 正面词典 | 正面情感词汇集合 |
| 负面词典 | 负面情感词汇集合 |
| 程度词典 | 程度副词集合（非常、极其等） |
| 否定词典 | 否定词集合（不、没等） |

**情感计算公式**：

```
情感得分 = Σ(正面词权重) - Σ(负面词权重) × 否定因子 × 程度因子

否定因子 = -1 (若存在否定词)
程度因子 = 程度词权重 (若存在程度词，默认1.0)
```

**情感分类阈值**：

| 情感得分 | 分类 |
|----------|------|
| > 0.1 | 正面 |
| < -0.1 | 负面 |
| [-0.1, 0.1] | 中性 |

#### 4.3.2 情感分析参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| positive_threshold | 0.1 | 正面情感阈值 |
| negative_threshold | -0.1 | 负面情感阈值 |
| use_negation | true | 是否处理否定词 |
| use_degree | true | 是否处理程度词 |

### 4.5 代码分析算法

#### 4.5.1 圈复杂度计算

**计算公式**：

```
Cyclomatic Complexity = E - N + 2P

E = 图中边的数量
N = 图中节点的数量
P = 连通分量数（通常为1）
```

**复杂度等级**：

| 复杂度值 | 等级 | 说明 |
|----------|------|------|
| 1-10 | 低 | 代码逻辑简单 |
| 11-20 | 中 | 代码有一定复杂度 |
| 21-50 | 高 | 代码复杂 |
| > 50 | 很高 | 代码难以维护 |

#### 4.5.2 代码相似度检测

**算法**：基于Jaccard相似度的代码相似度计算

```
Similarity = |A ∩ B| / |A ∪ B|

A = 代码A的token集合
B = 代码B的token集合
```

### 4.6 分词算法

#### 4.6.1 中文分词

**算法选择**：

| 算法 | 特点 | 适用场景 |
|------|------|----------|
| 正向最大匹配 | 简单快速 | 实时性要求高 |
| 逆向最大匹配 | 准确率较高 | 精度要求高 |
| 双向最大匹配 | 最佳准确率 | 精度要求高 |
| HMM | 支持新词识别 | 开放领域文本 |
| CRF | 准确率最高 | 专业领域文本 |

#### 4.6.2 分词参数

## 5. 错误处理

### 5.1 错误码定义

| 错误码 | 错误类型 | 说明 |
|--------|----------|------|
| A001 | ANALYSIS_TYPE_UNSUPPORTED | 不支持的分析类型 |
| A002 | CONTENT_EMPTY | 内容为空 |
| A003 | CONTENT_TOO_LARGE | 内容过大 |
| A004 | ANALYSIS_TIMEOUT | 分析超时 |
| A005 | ANALYZER_INIT_FAILED | 分析器初始化失败 |
| A006 | ANALYSIS_FAILED | 分析执行失败 |
| A007 | RESULT_SERIALIZE_FAILED | 结果序列化失败 |
| A008 | CACHE_FAILED | 缓存操作失败 |
| A009 | MODEL_NOT_FOUND | 分析模型未找到 |
| A010 | INVALID_PARAMETERS | 分析参数无效 |

### 5.2 错误响应格式

```json
{
  "success": false,
  "error": {
    "code": "A003",
    "message": "内容过大，无法分析",
    "details": {
      "maxSize": "10MB",
      "actualSize": "25MB"
    }
  },
  "request_id": "req_abc123"
}
```

### 5.3 错误处理策略

| 错误类型 | 处理策略 |
|----------|----------|
| 内容为空 | 返回错误，不进行后续分析 |
| 内容过大 | 返回错误，提示内容大小限制 |
| 分析超时 | 返回部分结果（如有），记录错误日志 |
| 分析器异常 | 尝试使用备用分析器，记录错误日志 |
| 缓存失败 | 降级为非缓存模式，继续分析 |

### 5.4 重试机制

| 场景 | 重试次数 | 重试间隔 |
|------|----------|----------|
| 网络请求类分析 | 3次 | 1s, 2s, 5s (指数退避) |
| 模型推理 | 2次 | 1s, 3s |
| 缓存读取 | 1次 | 0s |

## 6. 配置说明

### 6.1 配置文件结构

```yaml
analysis:
  # 全局分析配置
  global:
    # 是否启用分析功能
    enabled: true
    
    # 默认分析超时时间（秒）
    timeout: 300
    
    # 最大内容大小（字节）
    max_content_size: 10485760
    
    # 默认分析语言
    default_language: "zh-CN"
    
    # 是否启用缓存
    enable_cache: true
    
    # 缓存过期时间（秒）
    cache_ttl: 3600

  # 分析器配置
  analyzers:
    # 文本分析器
    text:
      enabled: true
      keyword:
        algorithm: "tfidf"  # tfidf, textrank
        top_k: 10
        min_word_length: 2
      summary:
        algorithm: "extractive"  # extractive, generative
        summary_ratio: 0.2
      sentiment:
        algorithm: "dictionary"  # dictionary, ml
        use_negation: true
        use_degree: true

    # 文档分析器
    document:
      enabled: true
      extract_structure: true
      extract_metadata: true

    # 代码分析器
    code:
      enabled: true
      languages:
        - python
        - javascript
        - java
      security_scan: true
      complexity_analysis: true

  # 性能配置
  performance:
    # 并行分析的最大并发数
    max_concurrency: 5
    
    # 是否启用流式处理
    streaming: false
    
    # 内存限制（字节）
    memory_limit: 2147483648

  # 高级配置
  advanced:
    # 自定义词典路径
    custom_dict_path: "/path/to/dict"
    
    # 模型配置
    models:
      sentiment_model: "sentiment_model_path"
      ner_model: "ner_model_path"
    
    # 调试模式
    debug: false
```

### 6.2 环境变量配置

| 环境变量 | 说明 | 默认值 |
|----------|------|--------|
| ANALYSIS_ENABLED | 是否启用分析功能 | true |
| ANALYSIS_TIMEOUT | 分析超时时间 | 300 |
| ANALYSIS_MAX_SIZE | 最大内容大小 | 10485760 |

## 8. 性能优化

### 8.1 优化策略

| 优化策略 | 说明 | 适用场景 |
|----------|------|----------|
| 缓存优化 | 缓存分析结果和中间数据 | 重复内容分析 |
| 并行处理 | 多任务并行执行 | 批量分析 |
| 增量分析 | 仅分析变更部分 | 持续分析场景 |
| 采样分析 | 大数据量采样分析 | 超大规模内容 |
